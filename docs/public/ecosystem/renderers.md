# 生态渲染器

由社区和第三方维护的 A2UI 渲染器实现。

> NOTE
> 这些渲染器由各自作者维护，而不是由 A2UI 团队维护。请自行确认每个项目的兼容性、版本支持情况和维护状态。

> TIP
> 正在寻找**官方**的 A2UI React 渲染器？请参见 [`@a2ui/react`](https://www.npmjs.com/package/@a2ui/react) —— 由 A2UI 团队维护的核心 A2UI React 渲染器。

### 重点项目

**easyops-cn/a2ui-sdk**（`@a2ui-sdk/react`）是目前最成熟的社区 React 渲染器，已发布 11 个版本，使用 Radix UI primitives、Tailwind CSS 样式，并拥有独立的文档站点。它曾在 [A2UI 讨论区宣布](https://github.com/a2ui-project/a2ui/discussions/489)。如需官方 A2UI React 渲染器，请参见 [`@a2ui/react`](https://www.npmjs.com/package/@a2ui/react)。

**lmee/A2UI-Android** 填补了一个重要空白 —— 它是目前唯一的 Jetpack Compose 渲染器，支持 Android 5.0 及以上版本，提供 20+ 个组件，并支持数据绑定与可访问性。

**sivamrudram-eng/a2ui-react-native** 是目前唯一的 React Native 渲染器，通过单一代码库即可在 iOS 和 Android 上使用 A2UI。

**BBC6BAE9/a2ui-swift**（原名 **a2ui-swiftui**）是一个原生 SwiftUI 渲染器，支持 iOS、macOS、visionOS、watchOS 和 tvOS。它覆盖了全部 18 个基础 A2UI 组件，同时支持 v0.8 和 v0.9 双协议，包含 JSONL 流式传输，并使用 Observation 框架实现响应式更新。

**TanXudong-Vivo/A2UI-Android-Renderer** 是一个基于 Jetpack Compose 和 Material 3 构建的模块化 Android 渲染器，也是首个支持 A2UI v0.9 协议的 Android 实现。它完整实现了 13 个组件（包括通过 Coil 实现的 Image 加载）、来自 LLM token 流的流式渲染、基于 `path` 表达式和 `formatDate` 的数据绑定，以及一个可插拔的 Custom Catalog，用于注册额外的组件类型。其演示应用中包含与官方 `restaurant_finder` ADK 智能体的实时连接。

**AGenUI/AGenUI** 是面向 A2UI v0.9 的跨平台原生渲染器，通过共享的 C++ 核心支持 iOS、Android 和 HarmonyOS。它的设计目标是高性能、可扩展性和跨平台一致性。AGenUI 完整实现了 A2UI v0.9，并提供用于扩展 UI 组件和函数调用的运行时 API。它还引入了一个[自定义 Catalog](https://github.com/AGenUI/AGenUI/blob/main/agenui_catalog.json)，在 Basic Catalog 的基础上扩展了 Table、Carousel、Web、RichText 组件，以及用于实现更丰富外观与布局控制的 Styles 属性，同时与 A2UI 可扩展的 Catalog 模型保持一致。参见[组件演示](https://genui.amap.com/components)、对应的开源 [A2UI Generation Skill](https://github.com/AGenUI/AGenUI/tree/main/skills/a2ui-generation)，以及 [GitHub](https://github.com/AGenUI/AGenUI) 上的更多信息。

**lynx-family/lynx-stack**（`@lynx-js/genui/a2ui`）为 A2UI v0.9 提供了 ReactLynx 渲染器。它通过 `MessageStore` 消费经过校验的服务端到客户端 A2UI 消息，根据调用方提供的 catalog 渲染已获批的 ReactLynx 组件，并通过 `onAction` 转发生成的 UI Action。已发布的 `@lynx-js/genui` 包通过 `a2ui` 子路径导出 A2UI 渲染器，而 GenUI CLI 则支持在构建期生成 catalog 产物和 A2UI 系统提示词。

## 提交你的渲染器

如果你构建了 A2UI 渲染器，我们很愿意把它列在这里。

### 提交方式

1. **Fork** [a2ui-project/a2ui](https://github.com/a2ui-project/a2ui) 仓库
2. **编辑** 本文件（`docs/ecosystem/renderers.md`）—— 在社区渲染器表格中新增一行，填入渲染器名称、平台、npm 包（如有）、支持版本和源码链接
3. **提交 PR** 至 `a2ui-project/a2ui`，并简要描述你的渲染器
4. **在 [GitHub Discussions](https://github.com/a2ui-project/a2ui/discussions) 发帖** —— 让社区知道你做了什么！一个简短的演示视频会非常有帮助。

需要灵感？可以浏览仓库中的 **[community samples](https://github.com/a2ui-project/a2ui/tree/main/samples)**。这些示例覆盖 Angular、Lit 和基于 ADK 的智能体，是很好的起点。

### 什么样的社区渲染器更容易被采纳？

如果你的项目具备以下特点，就更有可能被接受并被他人使用：

- 有**公开源码**（最好开源，MIT 或 Apache 2.0 更佳）
- 明确说明**支持哪个 A2UI 规范版本**（v0.8、v0.9 或两者）
- 覆盖**核心组件**：文本、按钮、输入组件与基础布局
- 带有包含安装说明和最小示例的 **README**
- **持续维护中** —— 如果你已不再维护，请明确标记为 archived

社区渲染器并不需要已经达到生产可用水平才可以列出。实验性和早期项目也非常欢迎进入"值得关注的项目"部分。
