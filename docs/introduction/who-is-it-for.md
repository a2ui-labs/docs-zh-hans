# A2UI 适合谁？

适合那些希望为 AI 智能体构建丰富、可交互 UI 的开发者。

## 三类目标用户

### 1. 宿主应用开发者（前端）

你正在构建多智能体平台、企业助手，或者让智能体负责生成 UI 的跨平台应用。

**为什么用 A2UI：**

- 品牌可控：样式与设计系统由客户端掌控
- 多智能体：支持本地、远端与第三方智能体
- 安全：声明式数据，不执行代码
- 跨平台：覆盖 Web、移动端、桌面端
- 可互操作：开源协议，同一规范可对应多个渲染器

**开始阅读：** [客户端接入](../guides/client-setup.md) | [主题与样式](../guides/theming.md) | [自定义组件](../guides/authoring-components.md)

### 2. 智能体开发者（后端 / AI）

你正在构建能够生成表单、看板和交互式工作流的智能体。

**为什么用 A2UI：**

- 对 LLM 友好：结构扁平，易于增量生成
- 交互丰富：不止文本，还能覆盖表单、表格、可视化等
- 生成物而非工具：UI 是智能体输出内容的一部分
- 可移植：一份智能体响应可在所有 A2UI 客户端中工作
- 可流式：边生成边渲染，实现渐进式体验

**开始阅读：** [智能体开发](../guides/agent-development.md)

### 3. 平台构建者（SDK / 平台作者）

你正在构建智能体编排平台、框架，或 UI 集成层。

你是否需要把远端智能体接入自己的应用？

你是否需要把自己的智能体部署到你并不完全控制的其他应用中？

**为什么用 A2UI：**

- 标准协议：可与 A2A 及其他传输层互操作
- 可扩展：支持自定义组件目录
- 开源（Apache 2.0）

**开始阅读：** [社区](../ecosystem/community.md) | [路线图](../roadmap.md)

## 什么时候适合使用 A2UI

- ✅ **由智能体生成 UI** —— 这是它的核心用途
- ✅ **多智能体系统** —— 在跨信任边界场景下使用统一协议
- ✅ **跨平台应用** —— 一个智能体，对接多个渲染器（Web / 移动端 / 桌面端）
- ✅ **安全要求高** —— 声明式数据，不执行代码
- ✅ **品牌一致性** —— 样式控制权在客户端

- ❌ **静态网站** —— 直接用 HTML / CSS
- ❌ **纯文本聊天** —— 用 Markdown 就够了
- ❌ **与客户端未深度集成的远端小部件** —— 更适合 iframe，例如 [MCP Apps](../introduction/agent-ui-ecosystem.md)
- ❌ **需要快速把 UI 和智能体一起搭起来的应用** —— 更适合 [AG UI / CopilotKit](../introduction/agent-ui-ecosystem.md)

更详细的对比请参阅 [智能体 UI 生态](../introduction/agent-ui-ecosystem.md)。
